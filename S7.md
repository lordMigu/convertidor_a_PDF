Resumen de cambios (auth/config/security/user)
- Se agregaron endpoints de recuperación y restablecimiento de contraseña: `POST /api/v1/auth/password-recovery` y `POST /api/v1/auth/reset-password`.
- El flujo de recuperación es genérico (no revela si el email existe) y dispara envío de correo en background con un link al frontend.
- Se incorporaron utilidades de seguridad para tokens de reset: `generate_reset_token()` y `hash_reset_token()`, almacenando sólo el hash.
- Se cambió el esquema de hash de contraseñas a Argon2 para compatibilidad con Python 3.13.
- Se ampliaron configuraciones: `frontend_base_url`, `password_reset_token_expire_minutes` y variables de correo (`MAIL_*`).
- Se extendió el modelo `User` con campos de reset de contraseña (`password_reset_token_hash`, `password_reset_token_expires_at`, `password_reset_token_used_at`).

# Prompt para implementar y probar en frontend
Implementa el flujo de “Recuperar contraseña” y “Restablecer contraseña” consumiendo la API FastAPI:
1) Pantalla “Recuperar contraseña”
   - Formulario con campo email y botón “Enviar enlace”.
   - Llamar `POST /api/v1/auth/password-recovery` con JSON: `{ "email": "user@dominio.com" }`.
   - Mostrar siempre mensaje genérico de éxito: “Si el correo existe, enviaremos un enlace para restablecer tu contraseña.”
   - Manejar error 500 mostrando: “Configuración de correo no disponible”.
2) Pantalla “Restablecer contraseña”
   - Leer el `token` desde la URL: `/reset-password?token=...`.
   - Form con `new_password` y confirmación; validar mínimo 8 caracteres y que coincidan.
   - Llamar `POST /api/v1/auth/reset-password` con JSON: `{ "token": "...", "new_password": "..." }`.
   - En éxito: mensaje “Contraseña actualizada correctamente” y CTA para ir a login.
   - En error 400: “Token inválido o expirado”. En 403: “Usuario inactivo”.
Incluye un flujo de prueba manual con datos mock y captura de respuestas (status y body) para verificar el comportamiento.
